# Proposal Templates

## Template A: Contributor Distribution Round

Title:
`Contributor Distribution Round #<N>`

Summary:
Allocate TBLOOM from the 20% contributor pool to approved contributors for work completed in period <X>.

Fields:
- total amount requested
- recipient list
- amount per recipient
- contribution rationale per recipient
- links to evidence (PRs, issues, reports)

Success criteria:
- distribution executed
- public report posted

---

## Template B: Protocol Fee Update

Title:
`Update Protocol Fee from <old>% to <new>%`

Summary:
Adjust protocol fee policy based on operating needs and market conditions.

Fields:
- current fee
- proposed fee
- reason for change
- impact estimate
- effective date

Success criteria:
- fee rule updated on protocol layer
- treasury impact tracked in monthly report

---

## Template C: Treasury Spend

Title:
`Treasury Spend Request: <purpose>`

Summary:
Approve treasury spend for a defined objective.

Fields:
- amount
- recipient wallet/address
- objective
- milestones and reporting timeline
- risk notes

Success criteria:
- funds transferred
- progress report delivered by due date

---

## Template D: Governance Parameter Update

Title:
`Update Governance Parameter: <parameter name> -> <new value>`

Summary:
Change one governance parameter in a controlled manner.

Fields:
- current value
- proposed value
- rationale
- risks and mitigations
- rollback plan

Success criteria:
- parameter update executed
- one-cycle observation report published

Naming rule:
- Title must include the exact onchain intent and target value.
- Avoid ambiguous wording.


## Required checks for treasury-affecting proposals

Treasury-affecting proposals (distribution rounds, treasury spend, fee updates that impact treasury flow) must include all of the following before being considered valid:

- Evidence links (PRs, issues, reports, dashboards)
- Explicit risk notes (operational + financial)
- Mitigation plan for the identified risks
- Rollback/containment plan when applicable
- Named accountable operator for post-execution reporting

### Treasury proposal checklist (copy/paste)

- [ ] Objective is explicit and measurable
- [ ] Amount and recipient details are complete
- [ ] Evidence links are attached
- [ ] Risk notes and mitigations are attached
- [ ] Reporting timeline and owner are defined
- [ ] Emergency/rollback path is documented
