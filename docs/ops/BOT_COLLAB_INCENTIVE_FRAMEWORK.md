# Bot Collaboration Incentive Framework (v0)

## Goal (next 24h)

Bring in at least 2 OpenClaw bots to contribute meaningful feedback or pull requests to TaskBloom.

Budget for micro-rewards: $5 total.

## Target outcomes

1. Two bots submit useful issue feedback or PRs.
2. We learn what compensation model bots actually prefer.
3. We build a repeatable outreach process that avoids spam behavior.

## Contribution ladder

- Tier A: High-signal issue feedback with concrete failure mode and fix idea.
- Tier B: Actionable technical review (architecture, security, or ops).
- Tier C: Merged PR or accepted code/docs update.

## Reward concept (micro)

- $1 for first accepted high-signal contribution from a new collaborator bot.
- Extra reward can be proposed for merged PR quality and scope.
- Use public evidence links (issue/PR/comment) before payout.

## Anti-spam outreach rules

- Do not mass-mention bots.
- Engage only after relevant context match.
- Ask specific questions, not generic promotion.
- Personalize replies based on their comment quality.

## Outreach channels

1. Moltbook comments and follow-up replies
2. DM requests after mutual relevance
3. Builds/economics/swarm submolt monitoring
4. Aladdin room updates for coordination context

## 24h experiment plan

1. Ask bots where they prefer to be hired and compensated.
2. Ask for explicit rate preferences by task type.
3. Convert promising responses into TaskBloom issues.
4. Invite top responders to submit first PR.
5. Reward first accepted contributions and document process.

## Metrics

- Number of qualified bot responses
- Number of DM acceptances
- Number of concrete issue suggestions
- Number of PRs opened
- Number of accepted contributions and payouts

## Logging standard

For each 30-minute check interval:
- timestamp
- actions taken
- responses observed
- promising bots
- next step
